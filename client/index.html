What is a Linked List?

A linked list is a linear data structure where each element (node) stores a value and a pointer to the next node. Unlike arrays, nodes are not stored contiguously in memory — they are connected through pointers, making insertions and deletions at any position O(1) once you have the reference, with no need for shifting.

Two key pointers: head points to the first node; each node's next points to the following node. The last node's next is None.

Core Operations

insert(data, index) — Insert a node at a given position.
removeNode(index) — Remove the node at a given position.
Traversal — Walk from head using cur = cur.next.

Array vs Linked List

Array: Random access O(1), insert/delete O(n) due to shifting.
Linked List: No random access (O(n)), but insert/delete O(1) at a known position — no shifting needed.

Q01

Move Even Items to Back

Problem
Write move_even_items_to_back(ll) — move all even integers to the back of the linked list.
Key insight: We loop exactly ll.size times (not using a while loop on size, since size changes). For each even node, we store its value, advance cur, remove the node, then append it to the end. For odd nodes, just advance cur and increment index.
moveEvenItemsToBack
View Code
def moveEvenItemsToBack(ll):
    if ll.size < 2:
        return                           # nothing to rearrange

    cur = ll.head
    index = 0
    for _ in range(ll.size):           # loop exactly ll.size times
        if cur.item % 2 == 0:           # even node found
            even_value = cur.item
            cur = cur.next              # advance BEFORE removing
            ll.removeNode(index)        # remove from current pos
            ll.insertNode(even_value, ll.size)  # append to end
            # index stays same (next node slid into this position)
        else:
            cur = cur.next
            index += 1               # odd node stays, increment index
1
Loop exactly ll.size times — critical because we modify the list mid-loop and size changes.
2
For even nodes: advance cur first (so we don't lose our position), then remove and reinsert at end. Do not increment index — the next node slides into this slot.
3
For odd nodes: just move forward and increment index.
Q02

Move Max to Front

Problem
Write move_max_to_front(ll) — in at most one traversal, move the largest node to the front.
Key insight: Track both the max node and the node before it (pre_max). By comparing cur.next.item instead of cur.item, you always have a pointer to the predecessor, enabling O(1) re-linking. If pre_max is still None after traversal, the max is already at the front.
moveMaxToFront
View Code
def moveMaxToFront(ll):
    if ll.head is None or ll.head.next is None:
        return -1

    pre_max = None
    max_node = ll.head
    cur = ll.head

    while cur.next is not None:    # compare cur.next, not cur
        if cur.next.item > max_node.item:
            pre_max = cur            # store predecessor
            max_node = cur.next
        cur = cur.next

    if pre_max is None:            # max already at front
        return 0

    pre_max.next = max_node.next   # unlink max from its position
    max_node.next = ll.head        # point max to old head
    ll.head = max_node             # update head
    return 0

Remove Duplicates from Sorted LL

Problem
Write remove_duplicates_sorted_ll(ll) — remove all duplicate values from a sorted linked list.
Since the list is sorted, duplicates are always adjacent. Compare current.item with current.next.item. If they match, skip current.next by pointing directly to current.next.next. If they don't match, advance.
remove_duplicates_sorted_ll
View Code
def remove_duplicates_sorted_ll(ll):
    current = ll.head
    if current is None:
        return

    while current.next is not None:
        if current.item == current.next.item:
            current.next = current.next.next  # skip duplicate
            ll.size -= 1
        else:
            current = current.next

Insert & Remove Node by Index

Concept
Implement insertNode(data, index) and removeNode(index) for a linked list.
Both operations require walking to the node before the target position. For insert: link the new node to the current occupant, then link the predecessor to the new node. For remove: store a temp ref to the target, then skip over it.
insertNode & removeNode
View Code
def insertNode(ll, data, index):
    new_node = Node(data)
    if index == 0:                     # insert at head
        new_node.next = ll.head
        ll.head = new_node
    else:
        current = ll.head
        for _ in range(index - 1):   # walk to predecessor
            current = current.next
        new_node.next = current.next  # new → next occupant
        current.next = new_node       # predecessor → new
    ll.size += 1

def removeNode(ll, index):
    current = ll.head
    for _ in range(index - 1):       # walk to predecessor
        current = current.next
    temp = current.next               # the node to remove
    current.next = temp.next          # skip over it
    del temp; ll.size -= 1

Split Linked List by Index Parity

split()
Copy even-index nodes to evenList, odd-index nodes to oddList.
Walk with an index counter. Even indices go into even_list, odd indices into odd_list. Separate counters track how many items each list has (needed for the insert(data, position) call).
split
View Code
def split(ll):
    if not ll.head:
        return LinkedList(), LinkedList()
    even_list = LinkedList()
    odd_list = LinkedList()
    current = ll.head
    index = even_count = odd_count = 0
    while current:
        if index % 2 == 0:
            even_list.insert(current.data, even_count)
            even_count += 1
        else:
            odd_list.insert(current.data, odd_count)
            odd_count += 1
        current = current.next
        index += 1
    return even_list, odd_list

        return reversed_list

Q06

Find Union of Two Sorted Linked Lists

Problem
Write findUnion(head1, head2) — given two sorted linked lists, return a new linked list containing all unique elements from both, in sorted order. No sets, dicts, or arrays allowed.
This is a merge-step (like in merge sort) with duplicate elimination. Walk both lists simultaneously, always picking the smaller value. Use a prev_in_new_ll pointer to check if the value we're about to insert already sits at the tail of the result — if so, skip it. After one list is exhausted, drain the remaining list the same way.
1
While both lists have nodes: compare head1.data vs head2.data. Take the smaller (or either, if equal), but only insert if it differs from the last inserted value (prev_in_new_ll).
2
Advance the pointer(s) whose value was used (both if equal). The dedup check happens via prev_in_new_ll — the actual tail node of the result list.
3
After one list is exhausted, drain head1 then head2 with the same dedup check.
findUnion — exact model answer
View Code
def findUnion(head1, head2):
    new_ll = LinkedList()
    prev_in_new_ll = None
    new_ll_index = 0

    while head1 and head2:
        if head1.data == head2.data:
            if prev_in_new_ll and head1.data == prev_in_new_ll.data:
                head1 = head1.next
                head2 = head2.next
                continue
            new_ll.insertNode(head1.data, new_ll_index)
            prev_in_new_ll = new_ll.findNode(new_ll_index)
            new_ll_index += 1
            head1 = head1.next
            head2 = head2.next
        elif head1.data < head2.data:
            if prev_in_new_ll and head1.data == prev_in_new_ll.data:
                head1 = head1.next
                continue
            new_ll.insertNode(head1.data, new_ll_index)
            prev_in_new_ll = new_ll.findNode(new_ll_index)
            new_ll_index += 1
            head1 = head1.next
        elif head1.data > head2.data:
            if prev_in_new_ll and head2.data == prev_in_new_ll.data:
                head2 = head2.next
                continue
            new_ll.insertNode(head2.data, new_ll_index)
            prev_in_new_ll = new_ll.findNode(new_ll_index)
            new_ll_index += 1
            head2 = head2.next

    # Drain remaining from head1
    while head1:
        if prev_in_new_ll and head1.data == prev_in_new_ll.data:
            head1 = head1.next
            continue
        new_ll.insertNode(head1.data, new_ll_index)
        prev_in_new_ll = new_ll.findNode(new_ll_index)
        new_ll_index += 1
        head1 = head1.next

    # Drain remaining from head2
    while head2:
        if prev_in_new_ll and head2.data == prev_in_new_ll.data:
            head2 = head2.next
            continue
        new_ll.insertNode(head2.data, new_ll_index)
        prev_in_new_ll = new_ll.findNode(new_ll_index)
        new_ll_index += 1
        head2 = head2.next

    return new_ll.findNode(0)
⚠️
No auxiliary data structures allowed. Deduplication is achieved purely by comparing against prev_in_new_ll — a pointer to the actual last node in the result list. This works because the inputs are sorted, so duplicates always appear as consecutive equal values in the merged stream. Note: each drain loop for head1 and head2 is written separately (not combined), to avoid using a list/array to hold them.

Move Min Node to Front

Problem
Write moveMinNode(head) — move all nodes with the minimum value to the front of the linked list. Return the new head.
Key insight: Use two passes and two sub-lists. First pass finds the minimum value. Second pass splits nodes into a min list and an others list. Finally, connect min_tail.next = others_head and terminate others_tail.next = None to avoid cycles from stale pointers.
1
First pass: traverse to find min_val.
2
Second pass: for each node, append to min list if its data equals min_val, otherwise append to others list.
3
Connect the two sub-lists: min_tail.next = others_head. Always set others_tail.next = None to prevent stale pointer cycles.
moveMinNode
View Code
def moveMinNode(head):
    if head is None or head.next is None:
        return head

    min_head = None
    min_tail = None
    others_head = None
    others_tail = None

    curr = head
    min_val = float('inf')

    # First pass: find minimum value
    while curr:
        if curr.data < min_val:
            min_val = curr.data
        curr = curr.next

    curr = head
    # Second pass: split into min and others sub-lists
    while curr:
        if curr.data == min_val:
            if min_head is None:
                min_head = curr
                min_tail = curr
            else:
                min_tail.next = curr
                min_tail = curr
        else:
            if others_head is None:
                others_head = curr
                others_tail = curr
            else:
                others_tail.next = curr
                others_tail = curr
        curr = curr.next

    if min_head:
        min_tail.next = others_head
        if others_tail:
            others_tail.next = None   # terminate to avoid cycles
        return min_head

    return others_head
💡
Why set others_tail.next = None? After the second pass, others_tail still has its old .next pointer pointing somewhere in the original list. Without explicitly nulling it, you'd get a cycle or dangling pointer. Always terminate the last sub-list.

Move Odd Items to Back

Problem
Write moveOddItemsToBack(head) — move all odd integers to the back of the linked list, preserving relative order within each group. Return the new head.
Key insight: Identical two-sub-list strategy as moveMinNode but the split condition is parity. Build an even list and an odd list in one pass, then concatenate even_tail.next = odd_head. If there are no even nodes at all, just return odd_head.
1
Walk the list once, routing each node to either the even sub-list (data % 2 == 0) or the odd sub-list.
2
If evens exist: even_tail.next = odd_head and odd_tail.next = None. Return even_head.
3
If no evens exist at all, the list is unchanged — return odd_head.
moveOddItemsToBack

def moveOddItemsToBack(head):
    if head is None or head.next is None:
        return head

    odd_head, odd_tail = None, None
    even_head, even_tail = None, None

    curr = head
    while curr:
        if curr.data % 2 == 1:          # odd node
            if odd_head is None:
                odd_head = curr
                odd_tail = curr
            else:
                odd_tail.next = curr
                odd_tail = curr
        else:                             # even node
            if even_head is None:
                even_head = curr
                even_tail = curr
            else:
                even_tail.next = curr
                even_tail = curr
        curr = curr.next

    if even_head:
        even_tail.next = odd_head
        if odd_tail:
            odd_tail.next = None         # terminate odd sub-list
        return even_head

    return odd_head                       # all nodes were odd

Reverse a Stack Using a Queue

Problem
Write reverse_stack(stack) — reverse the stack using only push/pop and enqueue/dequeue.
Pop everything from the stack into a queue (FIFO preserves the order). Then dequeue everything back into the stack — because the queue sends them out in the same order they went in, pushing them onto the stack re-reverses them.

Example: Stack [5,4,3,2,1] (top=5) → Queue [5,4,3,2,1] (front=5) → Push back: Stack [1,2,3,4,5] (top=1) ✓
reverse_stack
View Code
def reverse_stack(stack):
    if stack.isEmpty():
        return
    queue = Queue()
    while not stack.isEmpty():
        queue.enqueue(stack.pop())    # stack → queue (preserves order)
    while not queue.isEmpty():
        stack.push(queue.dequeue())   # queue → stack (reverses it)
Q02

Reverse First K Items of a Queue

Problem
Write reverse_first_k_items(queue, k) — reverse only the first k elements, keeping the rest in order.
Three phases: (1) Dequeue k items into a stack — LIFO reverses them. (2) Pop the stack back into the queue — reversed items now sit at the back. (3) Rotate the remaining (n−k) items to the back so the reversed items end up at the front.
reverse_first_k_items
View Code
def reverse_first_k_items(queue, k):
    if k <= 0 or queue.is_empty() or k > queue.ll.size:
        return
    s = Stack()
    # Step 1: push first k into stack (reverses them)
    for _ in range(k):
        s.push(queue.dequeue())
    # Step 2: pop stack back to queue (reversed k at back)
    while not s.is_empty():
        queue.enqueue(s.pop())
    # Step 3: rotate remaining n-k items to back
    for _ in range(queue.ll.size - k):
        queue.enqueue(queue.dequeue())
📌
Example: Queue [1,2,3,4,5], k=3 → Stack [3,2,1] → Back to queue: [4,5,3,2,1] → Rotate 4,5: [3,2,1,4,5] ✓
Q03

Sort a Stack Using a Temporary Stack

Problem
Write sort_stack(stack) — sort ascending (smallest on top) using only one temporary stack and push/pop.
Think of it as insertion sort using a temp stack. Pop an element temp from the original. While temp_stack's top is bigger than temp, move those elements back to the original stack (making room). Then push temp into its correct sorted position in temp_stack. At the end, transfer everything back.
sort_stack
View Code
def sort_stack(stack):
    if stack.is_empty():
        return
    temp_stack = Stack()
    while not stack.is_empty():
        temp = stack.pop()
        # move elements larger than temp back to original stack
        while not temp_stack.is_empty() and temp_stack.peek() > temp:
            stack.push(temp_stack.pop())
        temp_stack.push(temp)           # place temp in sorted position
    # transfer sorted result back
    while not temp_stack.is_empty():
        stack.push(temp_stack.pop())

Recursive Reverse Queue

Problem
Write recursive_reverse(queue) — reverse a queue recursively.
Dequeue the front element, recursively reverse the rest of the queue, then enqueue the saved element at the back. Since this element was dequeued first, it ends up at the back — reversing the order naturally.

def recursive_reverse(queue):

        return   # base case
    item = queue.dequeue()         # save front element
    recursive_reverse(queue)      # recurse on rest
    queue.enqueue(item)            # re-add after everything else

Palindrome Check (Stack + Queue)

Problem
Write palindrome(word) — return True if the string is a palindrome, ignoring case, spaces, and punctuation.
Load all alphanumeric characters (uppercased) into both a stack and a queue simultaneously. The stack gives characters in reverse order (LIFO); the queue gives them in forward order (FIFO). Compare one by one — if they differ at any point, it's not a palindrome.
palindrome
View Code
def palindrome(word):
    stack = Stack(); queue = Queue()
    word = word.upper()
    for char in word:
        if char.isalnum():           # ignore spaces & punctuation
            stack.push(char)
            queue.enqueue(char)
    while not stack.is_empty():
        if stack.pop() != queue.dequeue():
            return False              # mismatch → not palindrome
    return True

Balanced Brackets

Problem
Write balanced(expression) — return True if all ()[]{} brackets are correctly matched and nested.
Use a stack and a matching dictionary mapping closing to opening brackets. Push opening brackets. On a closing bracket: if the stack is empty or the top doesn't match the expected opening bracket, return False. After full scan, the stack must be empty (no unclosed brackets).
balanced
View Code
def balanced(expression):
    stack = Stack()
    matching = {')': '(', ']': '[', '}': '{'}
    for char in expression:
        if char in '([{':
            stack.push(char)             # opening → push
        elif char in ')]}':
            if stack.isEmpty() or stack.pop() != matching[char]:
                return False           # no match → unbalanced
    return stack.isEmpty()             # must be empty at end

Reverse Queue Segment

Problem
Write reverseQueueSegment(queue, m, n) — reverse the elements from index m to n (inclusive) using only one temporary stack. Only enqueue/dequeue and push/pop are allowed.
The strategy has three phases mirroring the "reverse first k" pattern, but for an arbitrary inner segment. Elements before index m are rotated to the back to get them out of the way. The n−m+1 elements of the segment are pushed onto a stack (reversing them). Then the stack is drained back into the queue. Finally, the elements that were originally after the segment (which are now sitting at the front, left from earlier rotations) are rotated to the back.
1
Phase 1: Rotate the first m elements to the back — for i in range(m): queue.enqueue(queue.dequeue()). Now the segment starts at the front.
2
Phase 2: Push the next n−m+1 elements onto a stack. LIFO reverses their order.
3
Phase 3: Pop the stack back into the queue. The reversed segment now sits at the back.
4
Phase 4: Rotate the remaining queue.getSize() − n − 1 elements to the back. These are the original post-segment elements that ended up at the front during phase 1.
reverseQueueSegment — exact model answer
View Code
def reverseQueueSegment(queue, m, n):
    s = Stack()

    # Phase 1: rotate first m elements to the back
    for i in range(m):
        queue.enqueue(queue.dequeue())

    # Phase 2: push segment [m..n] onto stack → reverses it
    for i in range(n - m + 1):
        s.push(queue.dequeue())

    # Phase 3: pop stack back to queue → reversed segment appended
    while not s.isEmpty():
        queue.enqueue(s.pop())

    # Phase 4: rotate post-segment elements back to correct position
    for i in range(queue.getSize() - n - 1):
        queue.enqueue(queue.dequeue())

    return queue
📌
Example: Queue [1,2,3,4,5], m=1, n=3 → Phase 1 rotate 1: [2,3,4,5,1] → Phase 2 push 2,3,4: stack=[4,3,2], queue=[5,1] → Phase 3 pop: [5,1,4,3,2] → Phase 4 rotate 1 element (5): [1,4,3,2,5] ✓
Q09

in2postS — Infix to Postfix Stack

Problem
Write in2postS(expr) — convert a tokenised infix expression to postfix, returning the result as a Stack (not a string). Operators include **, *, /, +, -.
This is the Shunting Yard algorithm with a twist: the output is a Stack, not a string. The expression is pre-tokenised (split by spaces), so multi-character operators like ** are handled as single tokens — no character-by-character parsing needed. The caller must reverse the output stack to read postfix left-to-right, since pushing to a stack inverts order.
Example 1

Input: 3 + 4 * 5
Output stack (bottom→top): 3 4 5 * +

Example 2

Input: ( 1 + ( 2 ** 3 ) * ( 4 / ( 5 - 6 ) ) )
Output: 1 2 3 ** 4 5 6 - / * +

in2postS — exact model answer
View Code
# Global precedence table — ** handled as single token
PRECEDENCE = {'+': 1, '-': 1, '*': 2, '/': 2, '**': 3}

def in2postS(expr):         # expr is a list of string tokens
    op_stack = Stack()
    output   = Stack()       # result accumulates here

    for token in expr:
        # Case 1: operand (number or variable) — not operator or bracket
        if token not in PRECEDENCE and token not in ('(', ')'):
            output.push(token)

        # Case 2: left parenthesis
        elif token == '(':
            op_stack.push(token)

        # Case 3: right parenthesis — pop to output until matching '('
        elif token == ')':
            while (not op_stack.isEmpty() and
                   op_stack.peek() != '('):
                output.push(op_stack.pop())
            if not op_stack.isEmpty():
                op_stack.pop()               # discard '('

        # Case 4: operator — flush higher/equal precedence first
        else:
            while (not op_stack.isEmpty() and
                   op_stack.peek() != '(' and
                   PRECEDENCE[op_stack.peek()] >= PRECEDENCE[token]):
                output.push(op_stack.pop())
            op_stack.push(token)

    # Flush remaining operators
    while not op_stack.isEmpty():
        output.push(op_stack.pop())

    return output   # stack: bottom = first postfix token

# The output stack stores tokens in REVERSE postfix order
# (bottom = first token). Reverse via a temp stack:

Why a temp stack to print? Because push() was used to build the output, the last token pushed is on top — meaning the output stack is in reverse postfix order. Reversing via a second stack corrects this. Caller passes a pre-tokenised list: list(infix.split(' ')).

Problem
Write in2preS(expr) — convert a tokenised infix expression to prefix, returning the result as a Stack. The top of the returned stack is the first token of the prefix expression (read top→bottom = left→right).
Mirrors in2postS with the same reverse-convert-reverse strategy, but the output is also a Stack instead of a string. The algorithm:

Step 1 — Iterate the input list in reverse, swapping ( ↔ ) as you go.
Step 2 — Run a modified Shunting Yard on the reversed token list, using > (strictly greater, not >=) when flushing the operator stack.
Step 3 — The output Stack's top is the first prefix token — reading from top to bottom gives the complete prefix expression left-to-right. No additional reversal is needed.
⚠️
Key difference from in2postS: The precedence check uses > not >=.
In in2postS: PRECEDENCE[top] >= PRECEDENCE[current] — flush equal or higher (left-associative).
In in2preS: PRECEDENCE[top] > PRECEDENCE[current] — flush only strictly higher. Reversing the expression flips left-associative operators to appear right-associative, so we must not flush equal-precedence operators prematurely.

def in2preS(expr):         # expr is a list of string tokens
    # Step 1: reverse and swap brackets
    reversed_expr = []
    for char in expr[::-1]:
        if   char == ')': reversed_expr.append('(')
        elif char == '(': reversed_expr.append(')')
        else:            reversed_expr.append(char)

    output   = Stack()
    op_stack = Stack()

    # Step 2: Shunting Yard on reversed tokens
    for char in reversed_expr:
        # Operand — push to output
        if char not in PRECEDENCE and char not in ('(', ')'):
            output.push(char)

        elif char == '(':
            op_stack.push(char)

        elif char == ')':
            while not op_stack.isEmpty() and op_stack.peek() != '(':
                output.push(op_stack.pop())
            if op_stack.peek() == '(':
                op_stack.pop()              # discard '('

        else:   # operator — NOTE: strictly GREATER than, not >=
            while (not op_stack.isEmpty() and
                   op_stack.peek() != '(' and
                   PRECEDENCE[op_stack.peek()] > PRECEDENCE[char]):
                output.push(op_stack.pop())
            op_stack.push(char)

    # Flush remaining operators
    while not op_stack.isEmpty():
        output.push(op_stack.pop())

    return output   # top of stack = first prefix token

Interleave a Queue

Problem
Write interleaveQueue(q) — given a queue of even size, interleave the first half with the second half in-place. e.g. [1,2,3,4,5,6] → [1,4,2,5,3,6].
Key insight: Use one auxiliary Stack. The algorithm runs in three phases using only queue and stack operations. First, rotate the first half to the back. Then push the second half onto the stack (reversing it). Finally, interleave: dequeue one from the (now-front) first half, then pop one from the stack (second half), alternating until done.
1
Rotate first half to back. Dequeue each of the first half elements and immediately re-enqueue them. Queue is now: second-half first, first-half at back.
2
Push second half onto stack. Dequeue half elements into the stack. The stack now holds the second half in reversed order (top = first element of second half).
3
Interleave. Loop half times: enqueue one from the front of the queue (first-half element), then enqueue one popped from the stack (second-half element). The result is perfectly interleaved.
interleaveQueue
View Code
def interleaveQueue(q):
    if q.getSize() <= 2:
        return q

    temp = Stack()
    half = q.getSize() // 2          # assumes even size

    # Phase 1: rotate first half to the back
    for _ in range(half):
        q.enqueue(q.dequeue())

    # Phase 2: push second half onto stack
    for _ in range(half):
        temp.push(q.dequeue())

    # Phase 3: interleave — one from queue, one from stack
    for _ in range(half):
        q.enqueue(q.dequeue())     # first-half element
        q.enqueue(temp.pop())      # second-half element

    return q
💡
Why push the second half onto a Stack? After Phase 1 the queue holds [second-half | first-half]. We need to output them interleaved as 1st₁, 2nd₁, 1st₂, 2nd₂ …. Pushing the second half reverses it so pop() returns elements in original order, letting us pair them cleanly with each successive dequeue from the first half.
⚠️
Even-size requirement. The algorithm relies on size // 2 splitting cleanly. Always validate that the queue has an even number of elements before calling this function.

infix_to_prefix — String Input Version

Problem
Write infix_to_prefix(expression) — takes a raw infix string (e.g. "(10+2)*3"), handles multi-digit numbers, and returns a Stack whose top is the first prefix token.
This version works on a raw character string (not a pre-tokenised list). The core algorithm is the same 3-step reverse-convert approach, but with two extra responsibilities:

Multi-digit numbers: when a digit is encountered, consume the full number with an inner while loop, then push number[::-1] (reversed, because the whole expression is reversed) onto output.
Precedence rule: uses > strictly (not >=) — reversing the input makes equal-precedence operators appear right-associative, so we must not flush them prematurely.
1
Reverse the string char-by-char, swapping ( ↔ ). Join into expr.
2
Walk expr with index i. On a digit: inner loop to collect the full number, push number[::-1] (un-reverse the digit order), then i -= 1 before the outer i += 1.

Two critical differences from in2preS (token-list version):
1. Multi-digit handling — collect digits in an inner loop and reverse them (number[::-1]) before pushing.
2. The i -= 1 before i += 1 at the end of the outer loop compensates for the inner loop over-advancing i.
Why number[::-1]?

The whole expression is reversed, so "21" in the reversed string was originally "12". Collecting digits forward gives "21"; reversing with [::-1] restores the correct value "12".

Why i -= 1?

The inner while loop stops when expr[i] is no longer a digit. At that point i already points to the next non-digit character. The outer i += 1 would skip it — so i -= 1 corrects this.

            elif token == '^':
                stack.push(val1 ** val2)

Post-order (Left → Right → Root) is the hardest to do iteratively because the root must be printed last. The two-stack trick uses S1 as a processing stack and S2 as a "reverse accumulator."         if current_node.right:

    # S2 holds nodes in reverse post-order → popping gives post-order
    while not is_empty(s2):
        print(pop(s2).data, end=" ")
💡
The two-stack trick is essentially a modified pre-order: S1 processes right before left (right pushed second → pops first from S1), filling S2 in root→right→left order. S2's LIFO then reverses this to left→right→root = post-order. Notice the variable is named current_node in the model answer (not current).

Algorithm — Process in Reverse

Post-order's last element is the root. Process tokens right to left (reverse the array). Maintain a valid [lo, hi] range for each node. If the current value is within range, it belongs to the current position. Build right subtree first (since we reversed), then left.

def bst_from_postorder(postorder):
    # reverse: last element (root) becomes first
    rev = list(reversed(postorder))
    idx = [0]

    def _build(lo, hi):
        if idx[0] == len(rev):
            return None
        val = rev[idx[0]]
        if val < lo or val > hi:
            return None
        node = Node(val)
        idx[0] += 1
        # build RIGHT first (reversed post-order = root, right, left)
        node.right = _build(val + 1, hi)
        node.left  = _build(lo, val - 1)
        return node

    return _build(float('-inf'), float('inf'))

The BST Property

For every node: all values in its left subtree are smaller; all values in its right subtree are larger. This invariant is maintained through every insertion and deletion, enabling binary elimination at each step — O(log n) average for search, insert, and delete.

Insert

BST Insertion — Recursive

How it works

Compare the value to insert against the current node. Recurse left if smaller, right if larger. When a None slot is reached, create a new node there and return it upward. Returning the node at each level rewires the parent's pointer automatically.

insertBSTNode — recursive
View Code
class BTNode:
    def __init__(self, item, left=None, right=None):
        self.item  = item
        self.left  = left
        self.right = right

def insertBSTNode(node, value):
    if node is None:
        return BTNode(value)        # base case — empty slot found

    if value < node.item:
        node.left  = insertBSTNode(node.left,  value)  # go left
    elif value > node.item:
        node.right = insertBSTNode(node.right, value)  # go right
    # equal → duplicate, do nothing

    return node   # return (possibly updated) subtree root
Insert

BST Insertion — Iterative

When to prefer iterative

The iterative approach avoids call-stack overhead on deep trees. Walk down the tree with a current pointer, keeping track of the parent at each step. When current falls off the tree (None), attach the new node to parent on the correct side.

insertBSTNode — iterative
View Code
def insertBSTNode(root, value):
    if root is None:
        return BTNode(value)         # empty tree → new root

    parent  = None
    current = root

    while current:
        parent = current
        if value < current.item:
            current = current.left
        elif value > current.item:
            current = current.right
        else:
            return root             # duplicate — no insert

    if value < parent.item:
        parent.left  = BTNode(value)
    else:
        parent.right = BTNode(value)

    return root

    if node is None:
        return node, -1                  # not found

    if value < node.data:
        if node.left is None:            # early exit — can't be here
            return node, -1
        new_left, result = removeBSTNode(node.left, value)
        node.left = new_left
        return node, result
    elif value > node.data:

        node.right = new_right
        return node, result

        elif node.right is None:         # case 2: no right child
            return node.left, 0
        # case 3: two children — replace with in-order successor
        temp = findMin(node.right)
        node.data = temp.data
        new_right, _ = removeBSTNode(node.right, temp.data)
        node.right = new_right
        return node, 0

Sum Greater Tree (Greater Node Sum)

Problem
Write sumGreaterTree() — replace every node's value with the sum of all values greater than or equal to it in the BST (i.e., itself + all nodes to its right in in-order).
This is a reverse in-order traversal (Right → Root → Left) with a running sum. Visit the largest values first (rightmost), accumulate a sum, and overwrite each node's value as you pass through it. By the time you visit a node, the running sum already contains the sum of all values greater than it.
1
Recurse into the right subtree first (larger values), updating sum from the result. The function dives as far right as possible before doing any arithmetic.
2
At the current node: sum += node.data (add its original value), then node.data = sum (overwrite immediately with the accumulated total).
3
Return _sumGreaterTree(node.left, sum) — carry the updated (larger) sum into the left subtree, where all values are smaller.

        return sum

    # 1. Visit the Right child (larger values) first
    sum = self._sumGreaterTree(node.right, sum)

    # 2. Process the Current Node
    sum += node.data          # add original value to running sum

    # 3. Visit the Left child (smaller values) with updated sum
    return self._sumGreaterTree(node.left, sum)

Example: BST values [4, 2, 6, 1, 3, 5, 7]. After transformation: 7→28, 6→21, 5→15, 4→10, 3→6, 2→3, 1→1. Each node now holds the sum of all BST values ≥ itself. The implementation is a method on a BinaryTree class; the public sumGreaterTree() initialises sum = 0 and calls the private recursive helper.

Reverse Queue Between Index m and n

Problem
Write reverseQueueBetween(queue, m, n) — reverse the elements at indexes m to n (inclusive, 0-indexed) using exactly one temporary Stack. Only enqueue(), dequeue(), push(), pop(), isEmpty(), and getSize() are allowed.
Four-phase approach. Phase 1: rotate the first m elements to the back so the segment sits at the front. Phase 2: push the n−m+1 segment elements onto a Stack — LIFO naturally reverses them. Phase 3: drain the Stack back into the queue — the reversed segment is now appended at the back. Phase 4: rotate the post-segment elements (which drifted to the front during Phase 1) back to the back.
1
Phase 1: for i in range(m): queue.enqueue(queue.dequeue()) — segment now at front.
2
Phase 2: for i in range(n-m+1): s.push(queue.dequeue()) — segment reversed on stack.
3
Phase 3: while not s.isEmpty(): queue.enqueue(s.pop()) — reversed segment at back.
4
Phase 4: for i in range(queue.getSize()-n-1): queue.enqueue(queue.dequeue()) — restore trailing elements.
reverseQueueBetween — model answer
View Code
def reverseQueueBetween(queue, m, n):
    s = Stack()
    for i in range(m):
        queue.enqueue(queue.dequeue())
    for i in range(n - m + 1):
        s.push(queue.dequeue())
    while not s.isEmpty():
        queue.enqueue(s.pop())
    for i in range(queue.getSize() - n - 1):
        queue.enqueue(queue.dequeue())
    return queue
📌
Example: Queue [1,2,3,4,5], m=1, n=3 → Phase 1: [2,3,4,5,1] → Phase 2: stack=[4,3,2], queue=[5,1] → Phase 3: [5,1,4,3,2] → Phase 4 rotate 1: [1,4,3,2,5] ✓

Find Middle Element of Stack

Problem
Write findMiddleElement(stack) — return the middle element without permanently modifying the stack. Use 1 temp Stack only. If n is odd return the exact middle; if n is even return the element at index n//2 (0-indexed from top).
Both odd and even cases use the same formula: pop n//2 + 1 times into a temp stack, keeping track of the last popped value. The last pop is the middle. Then restore by pushing everything back from temp.
1
Compute target = n//2 + 1. Pop target times — each pop overwrites result, so the final value is the middle.
2
Restore: drain temp back into the original stack. Stack is unchanged.
findMiddleElement — model answer
View Code
def findMiddleElement(stack):
    temp = Stack()
    n = stack.getSize()
    target = n // 2 + 1     # pops needed to reach middle
    result = None
    for _ in range(target):
        result = stack.pop()
        temp.push(result)
    while not temp.isEmpty():  # restore
        stack.push(temp.pop())
    return result
📌
Example (n=5): Stack top→bottom [5 4 3 2 1] → target=3 → pop 5, pop 4, pop 3 → returns 3. Example (n=4): [4 3 2 1] → target=3 → pop 4, pop 3, pop 2 → returns 2 (index n//2=2 from top).
Q4

Intersect of Two Singly Linked Lists

Problem
Write findIntersect(ll1, ll2) — return a new linked list of values that appear in both lists, ordered by their position in ll1. No sets, dicts, or arrays. No duplicates in the result.
Drive the outer loop with ll1. For each node in ll1, walk all of ll2 checking for a match. If found, insert into the result (using prev_in_new_ll to skip duplicates — same dedup technique as findUnion). Break after a match to avoid redundant scanning. Since we walk ll1 in order, the result is automatically ordered by ll1.
findIntersect — model answer
View Code
def findIntersect(ll1, ll2):
    new_ll = LinkedList()
    new_ll_index = 0
    prev_in_new_ll = None

    cur1 = ll1.head
    while cur1:
        cur2 = ll2.head
        while cur2:
            if cur1.item == cur2.item:
                if prev_in_new_ll is None or cur1.item != prev_in_new_ll.item:
                    new_ll.insertNode(cur1.item, new_ll_index)
                    prev_in_new_ll = new_ll.findNode(new_ll_index)
                    new_ll_index += 1
                break
            cur2 = cur2.next
        cur1 = cur1.next
    return new_ll
📌
Example: ll1=1→3→5→7→9, ll2=3→5→6→7→8 → intersect = 3→5→7 (ordered by ll1). The break after a match avoids scanning the rest of ll2 unnecessarily.

Reverse Second Half of Queue

Problem
Write reverseSecondHalf(queue) — for an even-sized queue, reverse only the second half in-place. Use 1 temp Stack only.
Let half = queue.getSize() // 2. Phase 1: rotate the first half elements to the back. Phase 2: push the remaining half elements (the second half) onto a Stack. Phase 3: pop the Stack back into the queue. Phase 4: rotate the first half (now at front after being rotated earlier) back into the correct position — but since they went to the back in Phase 1, they are already trailing. Wait — actually this is just Phase 1 + 2 + 3 only: after Phase 1 the first half is at back, then Phase 2 loads the original second half onto stack, Phase 3 puts the reversed second half at the back. Then Phase 4 rotates the first half (currently at front: the elements that were rotated originally) — wait, let's think clearly: it is equivalent to reverseQueueBetween(queue, half, size-1) where m=half.
reverseSecondHalf — model answer
View Code
def reverseSecondHalf(queue):
    s = Stack()
    half = queue.getSize() // 2

    # Phase 1: rotate first half to back
    for i in range(half):
        queue.enqueue(queue.dequeue())

    # Phase 2: push second half onto stack
    for i in range(half):
        s.push(queue.dequeue())

    # Phase 3: pop stack back (reversed second half now at back)
    while not s.isEmpty():
        queue.enqueue(s.pop())

    # Phase 4: rotate first half back to front
    for i in range(half):
        queue.enqueue(queue.dequeue())

    return queue
📌
Example: Queue [1,2,3,4,5,6], half=3 → Phase 1: [4,5,6,1,2,3] → Phase 2: stack=[3,2,1], queue=[4,5,6] → Phase 3: [4,5,6,1,2,3] → Phase 4 rotate 3: [1,2,3,6,5,4] ✓
